# Part 1 시장성 및 TTS 품질 평가 보고서

기준일: 2026-05-01

## 결론

Part 1은 "한국형 아포칼립스 생존 인터랙티브 웹게임"으로 시장 테스트 가치가 있다. 국내 모바일 게임 시장은 여전히 크고, RPG 집중도가 높아 비RPG 생존 서사형 포맷은 차별화 여지가 있다. 다만 현재 상업성의 병목은 콘텐츠 양보다 `TTS 감정 품질`, `초반 인지 부하`, `텍스트-선택-소리의 리듬`이다.

현재 상태는 프로토타입/비공개 테스트 기준으로는 진입 가능하다. 공개 테스트 전에는 핵심 TTS 20개를 실제 청음 평가하고, 평균 8점 미만이면 35~40개 핵심 라인을 ElevenLabs v3 또는 Hume/Cartesia 후보로 재제작하는 것이 맞다.

## 현재 제품 포지션

- 장르: 모바일 우선 웹 기반 한국형 아포칼립스 생존 게임북
- 범위: Part 1 / CH01~CH05
- 핵심 경험: 이미지, MP3 TTS, BGM, SFX, CSS 글리치/전환 연출, 기한 실패 이벤트
- 제외 범위: 영상 본편
- 현재 QA 근거:
  - `safe`, `balanced`, `risky` 3개 롱런 루트 모두 CH05 결과 도달
  - soft-lock 0건
  - console/page/request error 0건
  - deadline consequence 이벤트 노출 확인
- 현재 TTS 런타임 상태:
  - `public/generated/audio/tts/P1`: MP3 144개
  - 총 용량: 약 19.93MB
  - WAV 런타임 배포 없음

## 시장성 판단

### 긍정 신호

- 국내 모바일 게임 시장은 2025년 상반기 약 24억 달러 규모로 여전히 큰 시장이다.
- 한국 시장은 RPG 집중도가 높다. RPG가 국내 모바일 매출에서 과도하게 높은 비중을 차지한다는 점은 정면 경쟁 시 위험이지만, 반대로 비RPG 서사형 생존 게임에는 차별화 포지션을 준다.
- 생존/재난/아포칼립스 소재는 모바일에서도 검증된 흡인력이 있다. 다만 대형 전략/수집 게임과 정면 경쟁하면 안 되고, "짧고 강한 웹 서사 경험"으로 포지셔닝해야 한다.
- 영상 없이 이미지+음성+효과음+CSS로 구성하면 토스 WebView와 모바일 웹에서 초기 배포 리스크를 낮출 수 있다.

### 위험 신호

- 텍스트 기반 게임은 초반 1~2분 안에 몰입을 못 만들면 이탈이 빠르다.
- TTS가 기계적으로 들리면 오히려 몰입을 깨뜨린다. 이 경우 음성을 줄이고 SFX/BGM/텍스트 리듬 중심으로 가는 편이 낫다.
- HUD 정보량, 시간/기한/부상/감염/소음/인벤토리까지 한 번에 보여주면 플레이어가 이야기보다 숫자를 먼저 보게 된다.
- AI 이미지 톤이 반복되면 "비슷한 폐허 사진 모음"처럼 보일 수 있다. CH별 공간 정체성과 감염체 실루엣을 더 분리해야 한다.

## 사용자 반응 예상

### 통할 가능성이 높은 사용자

- 한국 배경 재난/좀비물을 좋아하는 모바일 사용자
- 텍스트 어드벤처, 비주얼 노벨, TRPG식 선택지를 받아들이는 사용자
- 짧은 회차형 스토리, 다회차 분기, 수집형 기록을 좋아하는 사용자
- "싸워서 이기는 게임"보다 "살아남는 선택"에 반응하는 사용자

### 이탈 가능성이 높은 사용자

- 빠른 액션, 성장 수치, 직접 전투를 기대하는 사용자
- 긴 텍스트를 싫어하는 사용자
- TTS가 어색하면 바로 품질 낮다고 느끼는 사용자
- 선택 결과가 불명확하거나 갑작스럽게 죽는다고 느끼는 사용자

## TTS 품질 평가

현재 TTS는 파일 구성과 런타임 최적화는 통과했다. 문제는 파일 존재 여부가 아니라 `연기 품질`이다. 현재 사용자는 기존 TTS를 "기계음에 가깝고 분위기와 맞지 않는다"고 평가했다. 따라서 공개 테스트 전에 청음 평가가 필요하다.

### 144개 전체 자동 감사 결과

- 상세 산출물: `docs/ops/PART1_TTS_144_EVALUATION.md`
- 구조화 데이터: `docs/ops/PART1_TTS_144_EVALUATION.json`
- 대상: 144개
- MP3 존재: 144개
- 누락: 0개
- 총 재생 길이: 약 38.93분
- 총 용량: 약 27.41MB
- 자동 기술 OK: 144개
- 자동 기술 검토 필요: 0개
- 조치: 기존 기술 검토 대상 62개를 OpenAI `gpt-4o-mini-tts` 감정 지시 기반 MP3로 재생성했다.
- 판정: 런타임 배포 기술 기준은 통과했다. 남은 검증은 실제 청음 기반 감정 연기 점수다.

### 평가 대상 20개

| Chapter | 개수 | 기준 |
|---|---:|---|
| CH01 | 5 | 오프닝, 첫 무전, 조우 경고, 기한 실패, 결과 |
| CH02 | 4 | 브리핑, 시장/수로, NPC 대사, 결과 |
| CH03 | 4 | 유리 고층 진입, 동료 대사, 조우, 결과 |
| CH04 | 4 | 물류 라인, 안보경/한소명 대사, 기한 실패, 결과 |
| CH05 | 3 | 김아라, 공개 범위 선택, 최종 결과 |

### 10점 평가 기준

| 항목 | 가중치 | 통과 기준 |
|---|---:|---|
| 감정 연기 | 35% | 속삭임, 숨, 긴장, 망설임이 자연스러워야 함 |
| 한국어 자연스러움 | 25% | 억양, 띄어 읽기, 조사 처리가 어색하지 않아야 함 |
| 장면 분위기 | 20% | 무전/폐허/추격/피로감에 맞아야 함 |
| 기술 품질 | 10% | 잡음, 깨짐, 과도한 압축감이 없어야 함 |
| 런타임 적합성 | 10% | 길이, 볼륨, 파일 크기가 게임 흐름에 맞아야 함 |

### 판정 기준

- 평균 8.0 이상: 프로토타입/비공개 테스트 유지 가능
- 평균 8.0 미만: 핵심 20개부터 재제작
- 핵심 라인 중 6.5 미만 존재: 해당 라인은 즉시 재제작
- 상업용 목표: 평균 8.8 이상, 핵심 라인 7.5 미만 0건

## TTS 재제작 계획

### 1차: 핵심 20개 오디션

- 동일 대사로 `ElevenLabs v3`, `Hume`, `Cartesia`, 기존 OpenAI TTS를 비교한다.
- 동일 볼륨/길이로 맞춘 뒤 브라우저에서 실제 이벤트 진입 타이밍에 맞춰 듣는다.
- 단독 청음이 아니라 BGM/SFX 위에서 들어야 한다.

### 2차: 35~40개 핵심 라인 재제작

- CH01~CH05의 첫 진입, 조우, 기한 실패, 결과, 주요 NPC 대사만 우선 교체한다.
- 전체 144개 풀보이스 재제작은 아직 하지 않는다.
- 대체 불가능한 감정 대사만 비용을 써야 한다.

### 3차: 전체 재제작 여부 결정

- 공개 테스트에서 TTS 불만이 반복되면 나머지 라인을 재제작한다.
- 반대로 TTS보다 텍스트/SFX 몰입도가 더 좋으면 일부 라인은 음성 없이 유지한다.

| ElevenLabs v3 | 감정 연기와 캐릭터 대사 우선 후보. 다만 긴 문단에서 톤 흔들림 리스크가 있어 짧은 라인 단위 제작이 적합 |

| OpenAI gpt-4o-mini-tts | 비용/자동화 후보. 현재 감정 품질 불만이 있으므로 상업용 핵심 라인 기본값으로는 보류 |
