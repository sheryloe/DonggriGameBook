# PRN · Story 02 외곽 보급 루트

## 1. 개요

- `routeId`: `2`
- `version`: `0.2.0`
- `storyId`: `story-02`
- 테마는 여의도 외곽 보급 루트다.
- 핵심 감각은 `배차`, `제방`, `톨게이트 검문`, `교환 장터`, `중계창고`, `수문`, `임시 적치장`, `복귀 판정`이다.
- Story 01보다 짧은 시작점이 아니라, 실제로 눌러 볼 수 있는 2편의 첫 회차로 둔다.
- 시스템은 Story 01과 동일하게 `보급(supplies)`과 `소음(noise)` 두 값만 쓴다.

## 2. 플레이 목표

- 여의도 내부의 관료 질서가 외곽으로 나가면 어떻게 느슨해지는지 보여준다.
- 외곽에서의 공포는 좀비 자체보다 검문, 통행권, 화물 목록, 공제, 신뢰 손상에서 온다.
- 영화적 감각은 참고하되 장면 복제는 하지 않는다.
- `부산행`은 병목과 압박, `반도`는 외곽 회수 경제, `콘크리트 유토피아`는 거주 자격과 우선권 정치의 감각을 가져온다.

## 3. 실제 플레이 구조

- 직행 경로는 `/story/2/:nodeId`다.
- 시작 노드는 `dispatch`다.
- 노드 순서는 아래와 같다.

| 순서 | Node ID | 장면 |
| --- | --- | --- |
| 1 | `dispatch` | 보급 배차 |
| 2 | `floodwall` | 제방 진입 |
| 3 | `tollgate` | 외곽 검문선 |
| 4 | `market` | 교환 장터 |
| 5 | `relay` | 중계창고 |
| 6 | `sluice` | 수문 구간 |
| 7 | `depot` | 임시 적치장 |
| 8 | `return` | 복귀 판정 |
| 9 | `ending-clear` | 보급망 복구 |
| 10 | `ending-debt` | 공제 귀환 |
| 11 | `ending-quarantine` | 격리 보류 |

## 4. 엔딩 방향

- `ending-clear`: 보급망 복구, 정식 배차 가능
- `ending-debt`: 공제 귀환, 감시와 재배차 대기
- `ending-quarantine`: 격리 보류, 루트 실패

## 5. Bootstrap route arrays

- `clearRoute`: `[0, 0, 0, 0, 0, 0, 0, 0]`
- `debtRoute`: `[0, 0, 0, 0, 0, 0, 0, 1]`
- `quarantineRoute`: `[1, 1, 1, 1, 1, 1, 1, 2]`

## 6. 노드 설계 메모

- `dispatch`는 배차 문서와 개인 호출표가 처음 만나는 자리다.
- `floodwall`은 한강 제방과 서비스 도로의 압박을 보여준다.
- `tollgate`는 길이 곧 신분이 되는 검문선을 보여준다.
- `market`은 물자보다 신뢰가 먼저 계산되는 교환 경제를 보여준다.
- `relay`는 정식 접수와 비공식 재분배가 충돌하는 중계창고다.
- `sluice`는 수문 주기와 경보등이 사람의 속도를 결정하는 구간이다.
- `depot`은 임시 적치장에서 봉인과 누락이 동시에 벌어지는 구간이다.
- `return`은 회수량보다 목격담과 접촉 흔적이 중요해지는 최종 판정이다.

## 7. 비주얼 원칙

- 신규 파일은 늘리지 않고, 기존 Story 02 SVG 콜라주를 재사용한다.
- `sluice`는 제방/수문 계열 비주얼을, `depot`은 중계창고/재분배 계열 비주얼을 재사용한다.
- 화면은 문서, 도식, 영수증, 배차표, 검문표처럼 보여야 한다.
- 공식 캐릭터나 로고 직접 재현은 금지한다.

## 8. 검증 기준

- `dispatch`에서 시작해 `return`까지 URL 직행이 가능해야 한다.
- `ending-clear`, `ending-debt`, `ending-quarantine`의 세 분기가 모두 실제로 닫혀야 한다.
- 기본 안전 루트는 10분 내외의 체감 플레이타임을 확보해야 한다.
- 플레이어는 이름, 호출표, 거주 자격, 소음/보급 상태를 Story 02에서도 그대로 인지할 수 있어야 한다.
