# 지역 / 맵 재사용 규칙

## 목적

- Part 2~4 제작 시 모든 지역을 새로 만들지 않도록 재사용 규칙을 고정한다.

## 재사용 우선순위

1. 기존 구조를 다시 칠하고 연출만 바꾼다.
2. 연결 방식과 이동 비용을 바꾼다.
3. 동일 구조물에 시간/기후/침수 상태를 다르게 씌운다.
4. 그래도 부족할 때만 신규 구조를 만든다.

## 재사용 가능한 공간 패턴

- 방송/행정 통제실 -> `봉쇄선의 방`, `잔향 기지`
- 물류 벨트/창고 -> `연기 저장고`, `백색 야적장`
- 수로/수문/저지대 -> `침하 항만`, `파편 수문`
- 고층/수직 이동축 -> `철의 우회`, `외해 전초`
- 서버/인증실 -> `독도의 문`

## JSON 제작 규칙

- node shape는 기존 `chapter_event.schema.json` 구조를 유지
- `chapter_id`만 다르고 `node_type`, `travel_type`, `revisit_rule`은 최대한 기존 vocabulary 재사용
- 신규 파트도 `entry_node_id`, `exit_node_ids`, `quest_tracks`, `ui_profile` 포맷을 유지
- 이미지 키는 새로 만들어도 `bg_`, `portrait_`, `boss_`, `doc_` prefix 규칙 유지
